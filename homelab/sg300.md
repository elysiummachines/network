# Cisco SG300-52 Firmware Upgrade Guide

## Overview
The SG300 series is EOL, Cisco has pulled all firmware from their site. This guide covers the full upgrade path from factory firmware to the latest available version using archive.org mirrors.

**Upgrade Path:**
```
1.1.2.0 → 1.3.7.18 → bootloader 1.3.5.06 → 1.4.11.5
```

---

## Prerequisites

### Firmware Files
All files sourced from archive.org:
```
https://archive.org/download/sx-300-firmware-bootcode
```

Files needed:
| File | Purpose |
|------|---------|
| `sx300_fw_1.3.7.18.ros` | Intermediate firmware |
| `sx300_boot-13506.rfb` | Bootloader update |
| `Sx300-FW-1.4.11.5.bin` | Final firmware |

### TFTP Server (Linux/Debian)
Required for bootloader upload only.

```bash
sudo apt install tftpd-hpa -y
sudo cp sx300_boot-13506.rfb /srv/tftp/
sudo systemctl status tftpd-hpa
```

### SSH Access
Modern SSH clients reject the switch's old algorithms. Use these flags:
```bash
ssh -oKexAlgorithms=+diffie-hellman-group1-sha1 \
    -oHostKeyAlgorithms=+ssh-rsa \
    -oCiphers=+aes256-cbc \
    cisco@<SWITCH_IP>
```

Create an alias for convenience:
```bash
alias CISCONOW='ssh -oKexAlgorithms=+diffie-hellman-group1-sha1 -oHostKeyAlgorithms=+ssh-rsa -oCiphers=+aes256-cbc cisco@<SWITCH_IP>'
```

## Important Notes

### L2 vs L3 Mode
- Switching between L2 and L3 mode **wipes the entire configuration** — factory reset.
- Decide which mode you need **before** configuring anything.
- L3 mode required for proper LACP and inter-VLAN routing.
- Default is L2. Change under **Administration > System Settings**.

### Two Image Slots
The switch has two firmware slots. Uploading new firmware goes to the **inactive slot** - your current firmware is safe until you explicitly switch the active image and reboot.

## Step 1 - Intermediate Firmware (1.1.2.0 → 1.3.7.18)

**Method: Web UI (HTTP/HTTPS)**

1. Navigate to **Administration > File Management > Upgrade/Backup Firmware/Language**
2. Transfer Method: **via HTTP/HTTPS**
3. Save Action: **Upgrade**
4. File Type: **Firmware Image**
5. Choose File: `sx300_fw_1.3.7.18.ros`
6. Click **Apply** — wait for "Copy finished"
7. Navigate to **Active Image** > select image with 1.3.7.18 > Apply
8. Reboot:
```
SGSW01#reload
```
9. Verify:
```
SGSW01#show version
```
Boot version will still show old — that's expected, next step fixes it.

---

## Step 2 - Bootloader Update (1.1.0.6 > 1.3.5.06)

**Method: TFTP (required - web UI does not support boot code via HTTP)**

Ensure TFTP server is running and `sx300_boot-13506.rfb` is in `/srv/tftp/`.

From switch SSH:
```
SGSW01#copy tftp://<TFTP_SERVER_IP>/sx300_boot-13506.rfb boot
```

Reboot after completion:
```
SGSW01#reload
```

Verify boot version updated:
```
SGSW01#show version
```
Expected: `Boot version 1.3.5.06`

---

## Step 3 - Final Firmware (1.3.7.18 > 1.4.11.5)

**Method: Web UI (HTTP/HTTPS)**

1. Navigate to **Administration → File Management → Upgrade/Backup Firmware/Language**
2. Transfer Method: **via HTTP/HTTPS**
3. Save Action: **Upgrade**
4. File Type: **Firmware Image**
5. Choose File: `Sx300-FW-1.4.11.5.bin`
6. Click **Apply** — wait for "Copy finished"
7. Navigate to **Active Image** > select image with 1.4.11.5 > Apply
8. Reboot:
```
SGSW01#reload
```
9. Verify:
```
SGSW01#show version
```

## Post-Upgrade Configuration

### NTP (L3 mode - settings differ from L2)
1. **Administration > Time Settings > SNTP Unicast** > Enable SNTP Client Unicast
2. Add servers (By IP):
   - Primary: `<DC_IP>` (internal domain controller running chronyd)
   - Secondary: public fallback
3. **Administration > Time Settings > System Time**:
   - Enable Main Clock Source (SNTP Servers)
   - Time Zone: UTC-5 (CDT) or UTC-6 (CST)
   - Daylight Savings: Enable → USA

### Enable SSH
**Administration > System Settings > TCP/UDP Services** > Enable SSH, Disable Telnet

### Save Config
Always save after changes:
```
SGSW01#copy running-config startup-config
```
Or via web: **File Management > Copy/Save Configuration**

## Chrony NTP Server Setup (Linux DC)

If using an internal Linux DC as NTP source, ensure it serves time to the network:

```bash
# Check chrony is running
systemctl status chronyd

# Enable serving to local network
sudo sed -i 's/#allow 192.168.0.0\/16/allow LOCAL_IP_ADDRESS\/24/' /etc/chrony.conf
sudo sed -i 's/#local stratum 10/local stratum 10/' /etc/chrony.conf
sudo systemctl restart chronyd

# Verify
cat /etc/chrony.conf | grep -E "allow|local"
```

## Troubleshooting

### "SW code file is over sized" error
You skipped the intermediate firmware step. You must go through 1.3.7.18 first before jumping to 1.4.x.

### SSH key exchange errors
Use the full SSH flags documented above. This is a known issue with old Cisco firmware and modern OpenSSH clients.

### NTP showing Unknown / 1899 date
- Ensure Main Clock Source is **enabled** in System Time
- In L3 mode also enable **SNTP Client Unicast**
- Check firewall on DC allows UDP port 123

### L3 mode wiped my config
Expected behavior - always set L2/L3 mode first before any other configuration.
